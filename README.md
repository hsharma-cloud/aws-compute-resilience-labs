# AWS Compute Resilience Labs

## Overview
This repository consolidates multiple AWS compute labs demonstrating
high availability, performance scaling, and self-healing architectures.
The labs showcase resilient design patterns that improve uptime,
fault tolerance, and operational efficiency.

---

## Labs Included

### 1. Multi-AZ Deployment
Deploying EC2 instances across multiple Availability Zones to improve
fault tolerance and ensure application availability during zone failures.

**Key Concepts**
- High availability
- Fault tolerance
- Multi-AZ architecture

---

### 2. Performance Scaling
Optimizing EC2 capacity to maintain performance under varying workloads
while ensuring secure and efficient resource utilization.

**Key Concepts**
- Capacity planning
- Performance optimization
- Cost-aware scaling

---

### 3. Auto Scaling & Auto Healing
Implementing Auto Scaling Groups and health checks to automatically
replace unhealthy instances and maintain service continuity.

**Key Concepts**
- Self-healing systems
- Dynamic scaling
- Resilience engineering

---

## Security & Governance Considerations
- Least-privilege security group configurations
- Controlled network access
- Cost governance through right-sizing
- Resilient architecture aligned with business continuity goals

---

## Key Outcomes
- Improved application resilience and uptime
- Reduced downtime through automated recovery
- Optimized performance during demand fluctuations
- Enhanced operational efficiency and cost governance
