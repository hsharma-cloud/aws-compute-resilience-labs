# AWS Auto Scaling & Auto-Healing Architecture

## Overview
Configured scheduled scaling rules and auto-healing capabilities for an EC2 Auto Scaling group to maintain application availability and resilience during demand fluctuations.

## Architecture Components
- EC2 Auto Scaling group  
- Scheduled scaling policies  
- Health checks for instance monitoring and replacement  
- Security groups for controlled access  

## Resilience & Operational Principles
- Automated recovery through auto-healing  
- Proactive scaling to support predictable demand  
- Reduced downtime through continuous health monitoring  
- Secure access control for scaling resources  

## Key Outcomes
- Maintained application availability during peak usage  
- Improved resilience through automatic instance replacement  
- Enabled proactive capacity management  
- Reduced operational overhead through automation  

## Technologies Used
- EC2 Auto Scaling  
- Scheduled scaling policies  
- Health checks  
- Security Groups
