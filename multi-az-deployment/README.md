# AWS EC2 Multi-AZ High Availability Architecture

## Overview
Designed and deployed an application across multiple Amazon EC2 instances in separate Availability Zones to improve resilience, ensure fault tolerance, and support business continuity requirements.

## Architecture Components
- Amazon EC2 instances deployed in multiple Availability Zones  
- Elastic Load Balancing for traffic distribution  
- Security Groups for controlled access  
- Auto Scaling for dynamic capacity and resilience  

## Security & Resilience Principles
- High availability through multi-AZ deployment  
- Reduced single points of failure  
- Controlled network access using security groups  
- Automated scaling to maintain service availability  

## Key Outcomes
- Improved application uptime and reliability  
- Enhanced fault tolerance across Availability Zones  
- Established resilient infrastructure supporting business continuity  

## Technologies Used
- Amazon EC2  
- Elastic Load Balancer  
- Auto Scaling  
- Security Groups
